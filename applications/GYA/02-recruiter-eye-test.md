# Step 2: Recruiter Eye Test — GYA

**Role applied:** Desarrollador Python Semi Sr (871) + Desarrollador Fullstack Jr/Semi Sr (868)
**CV:** Project Leader / EM / TPM profile from Mercado Libre
**Recruiter context:** Empresa regional en Tandil, equipo chico/mediano, busca perfil técnico-operativo

---

## 7-Second Glance

**Lo que salta primero:**
- "Mercado Libre" — pesa muchísimo. Marca conocida, reconocible al instante.
- "Project Leader", "Engineering Manager", "Technical Program Manager" — títulos de seniority alta.
- "20,000+ RPM", "squad of 9" — escala grande.
- "4 promotions in 18 months", "70% reduction" — métricas contundentes.

**¿Qué señalizan los must-haves?**
- ❌ Python, SQL Server, Frontend, Dashboards — nada de esto salta a la vista en los primeros 7 segundos.
- ✅ Java, API, grandes volúmenes de datos, IA — se ven, pero enterrados.

**Veredicto 7s:** *"Este perfil es mucho más senior de lo que estamos buscando. Es líder, no desarrollador. Y no veo SQL Server por ningún lado."*

---

## 30-Second Deeper Scan

### El Hook (señal más fuerte)
**Mercado Libre + n8n/AI automation + manejo de grandes volúmenes de datos.** Un ex-ML con experiencia real en automation con IA es raro de encontrar para una empresa de Tandil. Si pueden capturarlo, tienen a alguien que vio escala y problemas reales.

### Red Flags / Weak Signals

| Flag | Severidad | Por qué |
|------|-----------|---------|
| **Sobrecalificación** | 🔴 Alta | Project Leader en ML de 9 personas aplicando a Semi Sr o Jr/Semi Sr. El recruiter va a pensar "se va a ir en 6 meses cuando encuentre algo mejor" |
| **Sin SQL Server** | 🔴 Alta | Es el skill #1 de ambas JDs. No tenerlo es casi eliminatorio para ATS. |
| **Sin Frontend** | 🟡 Media | JD 868 pide explícitamente sistemas web y aplicaciones de escritorio. CV es 100% backend. |
| **Sin Ingeniería en Sistemas** | 🟡 Baja | Piden "estudios avanzados en Ingeniería en Sistemas". Vos tenés HarvardX + edX, que es más moderno pero no es lo que piden. |
| **CV en inglés** | 🟡 Media | Las JDs están en español. Para una empresa de Tandil, el CV debería estar en español. |
| **Perfil muy "manager"** | 🟡 Media | Todas las bullets son de liderazgo, coordinación, mentoring. Para un rol de desarrollador, necesita bullets más técnicas. |

### Un cambio que aumentaría la probabilidad de shortlist

**Reframear de "Project Leader" a "Developer + Data + Automation"** — cambiar el header a algo como *"Desarrollador Fullstack | Python & Java | Automatización con IA"* y reescribir las bullets de ML para que suenen a logros de desarrollo, no de management.

---

## Verdict

| Criterio | ¿Pasa? |
|----------|--------|
| **¿Shortlist para Python Semi Sr (871)?** | 🟡 **Maybe** — si reframeamos el perfil a técnico y agregamos SQL Server + dashboards. El n8n/AI y Python/Java pueden salvarlo. |
| **¿Shortlist para Fullstack Jr/Semi Sr (868)?** | 🟡 **Maybe (borderline)** — el gap de frontend pesa más acá. Sin experiencia web frontend explícita, es difícil. |
| **¿Pasaría si el CV estuviera optimizado?** | ✅ Sí, con un buen reframing técnico y agregando los keywords faltantes. |

---

## Key Insight

El problema de fondo no es técnico — **es de framing**. Tu experiencia real incluye:
- **Python/Java** ✅ (EXO + ML)
- **Datos y análisis** ✅ (20k+ RPM, debugging engine, datos)
- **Automatización con IA** ✅ (n8n, AI copilots)
- **SQL** ✅ (MySQL — aunque no SQL Server)

Pero el CV actual lo cuenta como **historia de liderazgo**, no como **historia de desarrollo técnico**. Para estas JDs, necesitamos contar la misma historia pero con otro enfoque.
