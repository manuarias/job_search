## Executive Summary

CV único optimizado para cubrir dos JDs de la misma empresa en Tandil:
- **JD 871:** Desarrollador Python Semi Sr
- **JD 868:** Desarrollador Fullstack con base Backend (Jr/Semi Sr)

**Score final:** 79.5/100 (vs 59.5/100 original — mejora de +20 puntos)

### Cambios clave aplicados:
1. **Traducción a español** con keywords técnicos en inglés
2. **Reframing de perfil:** de "Project Leader" a "Desarrollador Fullstack"
3. **Dashboards agregados:** Datadog + Tableau (confirmado por usuario)
4. **Agentes IA:** reframeado desde n8n + AI copilots
5. **Python destacado** al inicio de la experiencia EXO
6. **Frontend básico** agregado en experiencia temprana (sin exagerar)

### Gaps remanentes:
- **SQL Server** — no se inventó. Se cubre con "bases de datos relacionales (MySQL, SQL)".
- **Frontend avanzado** — se menciona experiencia básica, no se exagera.

## Key Decisions Made

| Decisión | Fundamento |
|----------|------------|
| **No inventar SQL Server** | El usuario no lo usó. Ponemos "bases relacionales (MySQL, SQL)" que es semánticamente cercano. |
| **Reframear a desarrollador** | Las JDs piden perfil técnico, no de liderazgo. Las métricas se mantienen, solo cambia el framing. |
| **Frontend básico sin exagerar** | El usuario confirmó experiencia básica. Suficiente para no mentir pero cubrir el keyword. |
| **CV único para 2 JDs** | Son de la misma empresa, roles compatibles, misma ubicación. El superset de keywords funciona. |
| **Mentoring con menos peso** | Para rol de dev Semi Sr, el mentoring no es el foco principal. Se incluye una bullet resumida. |
