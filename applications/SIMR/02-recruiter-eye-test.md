# Step 2: Recruiter Eye Test — Simera Java Developer (SIMR)

**Role Context:** Senior technical recruiter, 10+ years hiring Java Developers. Reviews 200+ resumes weekly. Hiring manager must-haves: Java backend development, REST API & MySQL experience, Agile/CI/CD practices.

---

## 7-Second Glance

**What stands out immediately:**
- Title says "Project Leader | Software Engineering | Team Management" — **this is NOT a Java Developer resume.**
- The word "Project Leader" dominates the header and summary.
- "Non-coding" appears in the Technical Background section — **instant disqualifier for a hands-on dev role.**
- "10 years of experience" and "Mercado Libre" are strong brand signals.
- "20,000+ RPM" catches the eye as a scale metric.

**Verdict after 7 seconds:**
**NO.** This reads as a management candidate trying to downshift into development. The recruiter's finger is already on the "Reject" button.

---

## 30-Second Deeper Scan

**The Hook (strongest signal):**
Mercado Libre + high-scale systems (20,000 RPM) + 70% incident reduction. This proves the candidate has operated in serious engineering environments. The "Shadow Processing & Debugging Engine" sounds technically impressive.

**The Red Flag (weakest signal):**
The last 3 years are framed as pure leadership: "negotiating with stakeholders," "roadmap planning," "talent development." There is ZERO mention of writing code, shipping features, or technical contribution in the most recent role. The "Non-coding" label confirms the fear — this person hasn't been hands-on.

**One Change to Increase Shortlist Probability:**
**Reframe the Project Leader role as a "Hands-on Technical Lead" who still architects, debugs, and ships code while leading the squad.** Change the header to "Java Developer | Backend Engineering | High-Scale Systems" and lead with Java, REST APIs, and CI/CD in the summary.

**Pass to Hiring Manager?**
**Maybe.** If the hiring manager is desperate for Java backend talent and willing to bet on someone who was recently in leadership, maybe. But most hiring managers for a mid-level Java role will pass because they want someone who *wants* to code, not someone who *used to* code and then managed for 3 years.

**Key Insight About Narrative Framing:**
This CV suffers from an **identity crisis**. It wants to be a leadership CV but is being sent to a developer role. The candidate MUST choose: either apply to Engineering Manager roles (where this CV works well) or completely rewrite for developer roles. If applying as a developer, the Project Leader section needs to scream "technical decision-making, architecture, and tooling" rather than "stakeholders and roadmaps."
